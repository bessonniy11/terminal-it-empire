# TIB-1: Управление сотрудниками

Документы задачи **TIB-1** хранятся здесь. Для крупных фич всегда создаём отдельную папку вида `docs/<ID>-<slug>` с таким же набором файлов.

## Состав
- [user-flow.md](user-flow.md) — пользовательский сценарий (как всё работает глазами игрока).
- [spec.md](spec.md) — техническая спецификация: модель данных, алгоритмы, сохранения.
- [tasks.md](tasks.md) — пошаговый план реализации с чекбоксами и тестами.

## Правила
1. Сначала описываем user flow, затем spec, затем tasks.
2. Кодим только после согласования документов.
3. Любые серьёзные изменения обновляют соответствующие файлы в docs.
